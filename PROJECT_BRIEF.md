# MOA — 프로젝트 종합 브리프 (발표 대본 작성용)

> 이 문서는 발표 자료/대본을 작성하는 AI가 **MOA 프로젝트 전체를 한 번에 이해**하도록 만든 요약본입니다.
> 모바일 컴퓨팅 수업(3학년 1학기) 팀 프로젝트이며, "기획 → 백엔드 API → 모바일 앱 → 웹 → AI → 배포"까지 완성된 상태입니다.

---

## 1. 한 줄 소개

**MOA는 팀 프로젝트 협업을 한 앱에서 끝내는 "올인원 협업 플랫폼"입니다.**
프로젝트 관리 · 할 일 · 채팅 · 회의(AI 회의록) · 일정 조율 · 파일 드라이브 · 기여도 리포트를 하나로 묶었습니다.

## 2. 누구를 위한 것 / 어떤 문제를 푸나

- **대상**: 팀플하는 대학생, 소규모 협업 팀
- **문제**: 팀플을 하면 카톡(채팅), 노션(할 일), 구글 드라이브(파일), 언제될까(일정 조율), 녹음앱(회의)을 **따로따로** 쓴다 → 흩어지고 번거롭다. 그리고 "누가 얼마나 기여했는지"가 불투명하다.
- **해결**: 이 모든 걸 **프로젝트 단위로 한 곳에** 모으고, **AI가 회의록을 자동 생성**하고 **기여도를 분석**해준다.

## 3. 핵심 차별점 (발표에서 강조할 포인트)

1. **AI 회의록** — 회의 오디오를 올리면 STT로 전사 → 요약 → 키워드 → **액션 아이템(할 일)** 추출 → 바로 To-Do로 등록 가능. 화자별 발언 분석 + 멤버 자동 매칭.
2. **QR 출석** — 회의 시작 시 QR이 생성되고, 팀원이 스캔하면 출석 체크 + 지각 시간(late_seconds)까지 자동 기록.
3. **기여도 리포트** — 멤버별 할 일 완료율 · 회의 발언량 등을 집계하고 **AI가 기여도 코멘트/점수**를 생성.
4. **올인원** — 채팅·드라이브·일정조율·회의가 전부 프로젝트에 연동.
5. **멀티 플랫폼** — 모바일 앱(React Native)과 웹(React) 둘 다 같은 백엔드를 공유.

## 4. 기술 스택

| 구분 | 기술 |
|---|---|
| **모바일** | React Native + Expo (Expo Router), TypeScript |
| **웹** | React + Vite, TypeScript |
| **백엔드** | FastAPI (Python) |
| **DB/인증** | Supabase (PostgreSQL + Auth + Storage + Realtime) |
| **AI - STT** | Daglo(다글로) 비동기 STT API (한국어 회의 음성 전사) |
| **AI - 요약/분석** | OpenAI GPT-4o-mini (요약·키워드·액션아이템·기여도 코멘트) |
| **이메일** | Resend (회원가입/비밀번호 OTP) |
| **푸시 알림** | Firebase Cloud Messaging |
| **배포** | 백엔드: Railway (`https://moa-mobilecomputing-production.up.railway.app`) |

**AI 추상화**: 모든 LLM 호출은 `services/ai.py` 한 곳에 모아둠 → 나중에 Gemini/Groq 등 OpenAI 호환 API로 base_url만 바꿔 전환 가능.

## 5. 시스템 아키텍처 (한눈에)

```
[모바일 앱 (Expo/RN)]  ┐
                       ├──> [FastAPI 백엔드] ──> [Supabase: DB / Auth / Storage]
[웹 (React/Vite)]      ┘            │
                                    ├──> [Daglo STT] (회의 음성 전사)
                                    ├──> [OpenAI GPT] (요약/분석)
                                    ├──> [Resend] (이메일 OTP)
                                    └──> [Firebase] (푸시)

[Supabase Realtime] ──> 채팅 메시지 실시간 수신 (앱이 직접 구독)
```

- **인증**: Supabase Auth + JWT Bearer 토큰. 모든 보호 API는 `Authorization: Bearer <token>`.
- **채팅 하이브리드**: 메시지 전송/조회는 FastAPI, 실시간 수신은 Supabase Realtime 구독.
- **보안 메모**: 백엔드가 service-role로 DB 접근(RLS 우회) → 앱 레벨에서 소유권/멤버십 체크로 권한 관리.

## 6. 전체 기능 목록 (세부)

### 6-1. 인증 / 온보딩
- 이메일 회원가입 → **OTP 이메일 인증** → 토큰 발급
- 로그인 / 아이디 찾기 / 비밀번호 찾기(전부 OTP 기반)
- 온보딩: 이름 설정 → 소속(학교/학과/학번) 설정 → 완료
- 프로필 편집, 비밀번호/아이디 변경

### 6-2. 프로젝트
- 프로젝트 CRUD (이름·이모지·색상·상태·기간)
- 상태: active / upcoming / completed, 남은 일수(D-day) 자동 계산
- 멤버 관리(추가/역할 지정/삭제), 멤버별 역할(팀장/개발자/디자이너 등)
- **프로젝트 초대**: 친구를 초대하면 pending 상태 → 받은 사람이 **알림에서 수락(역할 선택)/거절**. (모바일·웹 모두 지원)

### 6-3. 할 일 (To-Do)
- 할 일 CRUD, 완료 토글
- 담당자 배정(프로젝트 멤버), 마감일/시작일, 난이도(하/중/상)
- 내 할 일 모아보기 + 프로젝트별 보기, 달력 뷰

### 6-4. 회의 (핵심 AI 기능) 🎙
- **회의 시작** → QR 코드 생성 (`moa-meeting:{id}`), 시작자 자동 출석
- **QR 출석**: 팀원이 스캔 → 출석 체크 + 지각 시간 자동 기록
- **출석 현황**: 참석자/불참자, 불참 사유 입력
- **오디오 업로드** → Daglo STT 전사 → GPT 요약 → 키워드 추출 → **액션 아이템(할 일 후보)** 추출
- **화자 분석**: 화자별 발언 시간/샘플, 멤버 자동 매칭(+수동 매칭 보정)
- **액션 아이템 → To-Do 등록** (제목·담당자 지정)
- 회의 종료 시 진행 시간/불참 사유 저장

### 6-5. 기여도 리포트 📊
- 멤버별: 할 일 완료수/전체, 기여도, 점수, 발언 시간
- 전체: 총 할 일/완료/완료율, 회의 수
- **AI 종합 코멘트** + 멤버별 AI 코멘트(GPT 생성)

### 6-6. 채팅 💬
- 1:1(direct) + 프로젝트 단체 채팅방
- 실시간 메시지(Supabase Realtime), 읽음 표시
- **파일/사진 전송**(이미지 썸네일, 파일 다운로드)
- **공지** 등록/조회, **투표**(다중 옵션, 실시간 집계)
- 방 이름 변경, 나가기
- **권한**: 공지/투표 삭제는 작성자 또는 프로젝트 방장만 가능

### 6-7. 드라이브 📁
- 프로젝트 폴더 / 개인 폴더 / 중첩 서브폴더
- 파일 업로드(한글 파일명 지원), signed URL 다운로드, 이동/이름변경/삭제
- **AI 자동 정리**: 파일들을 주제별 폴더로 자동 분류(GPT)

### 6-8. 일정 조율 (When2Meet) 🗓
- 프로젝트별 일정 조율 생성(날짜들 + 시간 범위)
- 멤버가 가능한 시간대 체크 → **모두 가능한 시간(best slots) 자동 계산**
- 응답자 수/현황 표시. 삭제는 작성자/방장만

### 6-9. 친구
- 아이디 검색 → 친구 요청 → 수락/거절, 친구 목록, QR로 친구 추가

### 6-10. 알림
- 할 일 마감 임박, 회의록 생성, 프로젝트 초대, 친구 요청을 동적으로 합성
- 푸시 알림(FCM) 토큰 등록

## 8. 엔지니어링 품질 포인트 (발표에서 "신경 썼다"는 근거)

발표 마무리/Q&A에서 쓸 수 있는, 실제로 개선한 항목들:

- **동시성 처리**: 회의 STT(수 분 소요)·파일 업로드 같은 무거운 작업을 별도 스레드풀로 위임 → 한 명이 회의록을 만드는 동안에도 **다른 사용자 요청이 멈추지 않음**.
- **권한 세분화**: 공지/투표/일정조율 삭제를 "작성자 또는 방장"으로 제한(서버에서 강제 + UI에서 버튼 숨김).
- **데이터 정합성**: 소속 정보 중복 저장 방지(업서트).
- **운영 가시성**: 환경변수 누락 시 기동 경고, 조용히 삼키던 예외에 로깅 추가.
- **멀티플랫폼 일관성**: 모바일/웹이 동일한 백엔드 스키마(필드명) 공유.

## 9. 추천 데모 시나리오 (발표 시연 흐름)

1. **로그인 → 대시보드**: 내 프로젝트 한눈에
2. **프로젝트 생성 + 친구 초대** → (다른 계정) 알림에서 **초대 수락**
3. **할 일 추가** + 담당자 배정
4. **회의 시작 → QR 표시 → 팀원 스캔 출석** → 녹음 → **AI 회의록(요약·키워드·액션아이템)** 생성 → 액션아이템을 **할 일로 등록**
5. **기여도 리포트**: 멤버별 기여도 + AI 코멘트
6. **채팅**: 파일 전송 + 투표
7. **일정 조율**: 모두 가능한 시간 자동 추천
8. (선택) **드라이브 AI 자동 정리**

> 데모용 테스트 계정: `testfriend / test1234` (친구/채팅/초대 시연용)

- 청중이 비개발자면 "문제(팀플 흩어짐) → 해결(올인원+AI)" 스토리 위주, 개발자면 아키텍처/AI 파이프라인/동시성 위주.
